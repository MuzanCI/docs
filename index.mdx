**Muzan CI** is a <Tooltip headline="What is continuous integration?" tip="Continuous integration (CI) is the practice of automating builds and tests on every commit." cta="Learn more" href="/how-it-works/overview">continuous integration</Tooltip> platform designed for FreeBSD and Linux packages.

<Steps>
  <Step title="1. Push changes" icon="git-pull-request-arrow">
    Trigger jobs whenever new commits are pushed to your repository.
  </Step>
  <Step title="2. Build and test" icon="check">
    Run on a wide range of FreeBSD and Linux images.
  </Step>
  <Step title="3. Debug problems with SSH" icon="terminal">
    Connect to your jobs with SSH to troubleshoot failures.
  </Step>
  <Step title="4. Distribute packages (Coming soon)" icon="box">
     Publish packages to your users with `pkg`, `apt`, `yum`, and `pacman`.
  </Step>
</Steps>

<h1 noAnchor>
  Expressive and maintainable
</h1>

Muzan CI uses a [configuration language](/how-it-works/pipelines) that is

- **Familiar**, with Python-like syntax
- **Expressive**, with native support for loops and conditionals
- **Maintainable**, with reusable functions, variables, and imports

<Tabs>
  <Tab title="Matrix strategy">
    <CodeGroup>
```python pipeline.py
images = ["freebsd:14.4-RELEASE", "freebsd:15.1-RELEASE"]
arches = ["amd64", "arm64", "i686", "armv7"]

for image in images:
  for arch in arches:
      job(
        name = "test-{}-{}".format(image, arch),
        image = image,
        arch = arch,
        steps = ["make build", "make test"],
      )
```
    </CodeGroup>
  </Tab>

  <Tab title="Conditional jobs">
    <CodeGroup>
```python pipeline.py
if BRANCH == "main":
    job(name="deploy", steps=["make deploy"])
```
    </CodeGroup>
  </Tab>

  <Tab title="Functions and imports">
    <CodeGroup>
```python pipeline.py
# Import `clang_job` function from the `job_defs.py` file.
load("job_defs.py", "clang_job")

clang_job(name="sqlite", srcs=["sqlite.c"])
```

```python job_defs.py
def clang_job(name: str, srcs: list[str]):
  # Debug build
  job(
    name = "{}-debug".format(name),
    steps = ["clang -g3 -Og {}".format(" ".join(srcs))],
  )

  # Release build with optimizations
  job(
    name = "{}-release".format(name),
    steps = ["clang -flto -O3 {}".format(" ".join(srcs))],
  )

```
    </CodeGroup>
  </Tab>

</Tabs>
